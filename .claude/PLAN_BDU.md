# 문제 정리 (현재 상태)
- 실제 강의(카드): **7개**
- 파싱 결과 카드: **14개** (2배로 나옴)
- 실제 미청취: **3개**
- 파싱 결과 미청취: **39개** (전부 다 보이는 수준으로 잘못 집계)

---

# 원인 1) 카드가 14개로 나오는 이유
페이지가 보통 아래처럼 동작함:
- **Grid 뷰 카드 리스트**
- **List 뷰 카드 리스트**
를 **둘 다 DOM에 만들어 두고**, 실제로는 `active` 붙은 것만 화면에 보여줌.

그런데 파싱할 때 `.card`를 넓게 잡으면:
- 보이는 카드(7) + 숨겨진 카드(7) = **14개**

✅ 해결 원칙:
- **반드시 `active`인 카드 리스트만 선택**한다.

### 권장 CSS Selector
- 가장 안전:  
  `div.card-list.active > div.card`

- (가능하면) 특정 영역까지 좁힘:  
  `#divCardList.card-list.active > .card`

> 주의: 사이트 구현이 엉성하면 `id="divCardList"`가 중복일 수 있음  
> 그럴 땐 `div.card-list.active > div.card`가 더 안전함

# 원인 2) 미청취가 39개로 나오는 이유
카드 내부 onclick JSON을 보면 핵심 필드가 따로 있음.

### 미청취 개수(정답 필드)
- `ATND_NOT_CNT` → 예: 3

### 39로 보이는 값(미청취가 아님)
- `TOT_TIME_DIST` → 예: 39  
  (전체 시간/분포/총량 성격의 값으로 보이며, 미청취 카운트가 아님)

즉, 파싱 코드가 미청취를 구할 때:
- `ATND_NOT_CNT`가 아니라
- **`TOT_TIME_DIST`를 읽고 있어서 39로 잘못 나옴**

✅ 해결 원칙:
- 미청취는 **`ATND_NOT_CNT` 또는 배지 숫자(em.badge)**로만 계산한다.
